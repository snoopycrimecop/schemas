Introduction

This document outlines the XML Schemas created by the Open Microscopy Environment. Detailed information on the schemas is available here.

Schemas

This has the schema name, the current version, a link to the XSD file, and a short description. Generated documentation for the schemas is available at: http://www.openmicroscopy.org/Schemas/Documentation/Generated/OME-2016-06/ome.html

Note: Some browsers will try to render XSD files when you view them. This can result in either a blank screen or unformatted text. Choose to either download the file or view the source.

Active Schemas

  • OME - June 2016, version 2 [ome.xsd] The main schema which defines the OME ontology for microscopy.

Legacy Schemas

  • AnalysisChain - September 2009, version 1 [AnalysisChain.xsd] The Analysis Chain schema. Analysis chains are how module outputs are connected to inputs of other modules.
  • AnalysisModule - September 2009, version 1 [AnalysisModule.xsd] The Analysis Module Library schema. Analysis modules are image analysis algorithms or, more generally, how information is generated in OME. Makes use of CLI and MLI schemas.
  • BinaryFile - January 2015, version 1 [BinaryFile.xsd] The Binary File schema used to describe a file location, or the location of a fragment within a file. It is combined with the main OME schema as of 2016-06.
  • CA - September 2009, version 1 [CA.xsd] The standard Custom Attribute schema.
  • CLI - September 2009, version 1 [CLI.xsd] The Command Line Interface sub-schema.
  • DataHistory - September 2009, version 1 [DataHistory.xsd] The Data History schema. Data history describes the data dependency in OME.
  • Editor - April 2008, version 1 [Editor.xsd] The Editor schema.
  • MLI - September 2009, version 1 [MLI.xsd] The Matlab Interface sub-schema.
  • OMERO - June 2011, version 1 [OMERO.xsd] The schema extensions used by OMERO. It is not used by the OME schema but provides a structure for data in StructuredAnnotation blocks used by OMERO.
  • ROI - January 2015, version 1 [ROI.xsd] The Region Of Interest schema. It is combined with the main OME schema as of 2016-06.
  • SA - January 2015, version 1 [SA.xsd] The Structured Annotation schema. It is combined with the main OME schema as of 2016-06.
  • SPW - January 2015, version 1 [SPW.xsd] The Screen/Plate/Well schema. It is combined with the main OME schema as of 2016-06.
  • STD - September 2009, version 1 [STD.xsd] The Semantic Type definition schema. Once a Semantic Type is defined, data for the defined type can be imported into the OME server.
